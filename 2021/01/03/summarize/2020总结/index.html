2020总结

发表于 2021-01-03
字数统计: 1.2k | 阅读时长 ≈ 3

关于 2020

2020 是不平凡的一年，这一年从疫情中起步开始，好像开始就在告诉我们今年这一年会步步都很艰难似的，但是啦，我却没想到的是，不仅仅是开始让我这么难过，就连快要结束的时候，也让我这么难受。不过啦，对比于现在，我个人感觉已经没关系啦。

但是 2020 中所有的不开心和不愉快已经难过，都已经过去了，现在开启的是新的一年，是2021年啦，这一年，一定要让自己能够成长到满足的地步啦。

​ 2020 中的问题，也是需要自己对自己的进行一次汇总啦，从错误和问题中吸取教训，让自己更好的成长，不枉对过去的一年所行啦.

2020 问题

​ 这一年， 赖 和 犹豫不决 是我最大的问题。 赖，变相的体现出了我的一种拖延心里，这种心里一定要在接下来的时间中来进行杜绝，绝对不能让它再发生了。 特别是在每个周末中，这种表现是更加明显的，宁愿躺在床上不知不觉的刷过了很多时间去看小视频什么的，也没有更好的利用起来，希望能减少这部分时间的支出， 这种个人感觉要从打心底中来进行杜绝，那就且看接下来的我是要成长为什么样子.

犹豫不决这个问题也很严重，接下来，要利用果断的决定来改变这个性格。不然以后面对什么样子的问题，都会有犹豫不决的心态，这样是肯定不行啦。 但凡觉得有用和自己打心底就喜欢的话，那就朝着要去做或者要去买的步伐去前进，虽然这个步伐可能前进到很慢，但是还是要前进。

在技术层面上，也是有很多没有完成的。 比如没有更好的去深入 Java 的源码 以及 相对应的 Spring的框架的源码，对于 Netty / Dubbo 等这些优秀的框架，那就更加不用说了，肯定也是没有更好的深入进去，这是肯定不行的。 而且没有达到自己，每年可以利用时间去业余发展一门爱好以及一门语言来深入进来学习，这个也是没有达到的，是真的好遗憾. 也不晓得自己乱七八槽的去做了一些什么事情，总是没有达到自己想达到的点，还是蛮难受。 对于学技术的时间把握上，确实是还不够，缺少很多，这点我自己也不是很舒服。

2021展望

​ 对于 2020 年的未完成以及遗憾，绝不带入到 2021来，所以从这儿开始就进行一刀切啦.

​ 那么 2021 上，我希望可以更好的锻炼自己，去学习更多的东西，比如摄影做饭，技术就不用说啦，这是每天都必须需要去坚持的，但凡有天不坚持，那都是不行的啦。

那么来说来自己对 2021 的要求，希望自己每个月可以写一下总结，对指定的年计划来进行细分，对分到对应的每个月，然后每个月又细分到每个星期或者每二个星期啦，总是就是要细分来完成自己的任务啦。

​ 也希望自己不定期的录制下视频来记录下自己的生活啦。

​ 一定要深入的学习一门新的编程语言，这个前提是在不影响自己最初最本质的语言前提下啦。

​ 对于 微服务 / DDD 等要深入层次的去学习去了解，而不是仅仅的停留在处理简单的业务上，要想居然能有这种东西出现，那么肯定是有先对应更让人想不到的东西，总之这方面是需要加强.

​ 对于 k8s , 2020 也就实现了简单的搭建，这肯定是不行的，2021一定要重拾起来更深入的学习啦.

​ 对于 大数据领域的知识，也是重拾起来学习下，看下大数据中的产品是怎么处理的大量的数据，又是怎么计算的，就算是基于内存的，那又是怎么实现的呢？我认为这些都是不可少的。 我希望能有机器可以实践出来是最好的啦.

​ 对于 源码 一定要继续深挖下去.

​

那么，我们一起来 2021 加油.
